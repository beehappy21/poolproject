## ตารางพร้อมใช้สำหรับสไลด์หรือประชุมภายใน

| หัวข้อ | กติกาที่ระบบใช้จริง | หมายเหตุ/ตัวอย่าง |
| --- | --- | --- |
| Trigger เริ่มคอม | เริ่มเมื่อ order เป็น `approved` | `pending` หรือจ่ายแล้วแต่ยังไม่ approved ยังไม่คิดคอม |
| ฐานคิดคอม | ใช้ `PV` | runtime ตีความ `1 PV = 1 บาท` |
| สกุลเงินจ่ายคอม | `บาท` | ใช้ `finalPayableAmount` เป็นยอดจ่ายจริง |
| Direct L1 | `50%` ของ PV | order `200 PV` ได้ `100 บาท` |
| Direct L2 | `50%` ของ PV | order `200 PV` ได้ `100 บาท` |
| Team 2-leg | `30%` ของขาที่อ่อนกว่า | ใช้เมื่อมี `2 ขาที่จ่ายได้` |
| Team 3-leg | `50%` ของผลรวม `2 ขาที่อ่อนกว่า` | ใช้เมื่อมี `3 ขาที่จ่ายได้` |
| Matching L1 | `5%` ของ team final payable | คิดจากยอดที่จ่าย team ได้จริงหลัง cap |
| Matching L2 | `5%` ของ team final payable | เช่นเดียวกับ L1 |
| Pool | ใช้ `100%` ของ approved sales PV ของวันนั้นเข้ากอง | runtime ตอนนี้ `poolRate = 1` |
| Cashback | ปิด | ไม่ใช่แผน active ตอนนี้ |
| Unilevel | ปิด | ไม่ใช่แผน active ตอนนี้ |
| Matrix | ปิด | ไม่ใช่แผน active ตอนนี้ |
| Daily cap | `10,000 บาท/วัน` | ใช้กับ `Team 2-leg / Team 3-leg` final payable ของ runtime ปัจจุบัน |
| ครบรอบ | คอมสะสม `>= 10,000 บาท` | นับจาก Direct, Team, Matching, Pool |
| Grace หลังครบรอบ | `3 วัน` | คอมใหม่ยังคำนวณ แต่ถูก hold |
| ซื้อซ้ำเพื่อเปิดรอบใหม่ | ซื้อเองและ order ต้อง `approved` โดยมี `PV > 0` | ต้องเกิดภายใน grace 3 วัน |
| cap ของรอบใหม่หลังซื้อซ้ำ | `< 200 PV = 5,000`, `>= 200 PV = 10,000` | ใช้ PV ของ order ซื้อซ้ำกำหนด cycle cap รอบใหม่ |
| Cycle cap ระดับ 1 | PV สะสม `< 200` ได้ cap `5,000` | ใช้ PV สะสม ไม่ใช้ยอดบาท |
| Cycle cap ระดับ 2 | PV สะสม `>= 200` ได้ cap `10,000` | ถ้าเริ่มต่ำกว่า 200 แล้วซื้อเพิ่มจนถึง 200 ระบบอัปเกรดรอบเดิมให้ |
| Carry PV | PV ส่วนเกินเปิดรอบถัดไปแบบรอได้ | ระบบจ่ายรอบเก่าก่อน |
| Promote รอบถัดไป | เมื่อรอบก่อนหน้าเต็ม cap รอบถัดไปถูกเลื่อนขึ้นมารับคอม | ผ่าน UAT แล้ว |
| เงื่อนไขเข้าพูลครั้งแรก | ซื้อเอง 1 order approved + direct 3 คน + direct ทั้ง 3 ซื้ออย่างน้อยคนละ 1 order | ใช้เป็น first gate |
| เงื่อนไขรอบถัดไปของ pool | ไม่ต้อง rebuild 3 direct buyers | ใช้ qualifying self-repurchase แทน |
| รหัสชวนสมัคร | ใช้ `referralCode` | ห้ามใช้ `memberCode` แทนใน flow ใหม่ |
| การเปิดขาแนะนำ | ถ้ายังไม่ครบ `L / M / R` อย่างละ 1 direct ระบบบังคับ `AUTO` | เติมขาที่ว่างก่อนจนมีครบ 3 ขา |
| ลิงก์ระบุขา | ใช้ `L / M / R` ได้เมื่อครบทั้ง 3 ขาแล้ว | ก่อนครบ 3 ขาไม่เปิดให้ใช้ลิงก์กำหนดขา |
| AUTO หลังครบ 3 ขา | ลงขาที่ `ไม่มีคะแนน PV` ก่อน หรือขาที่ `คะแนน PV รวมน้อยสุด` | ใช้กติกาเดียวกันทั้ง runtime และ referral link |
| รหัสสมาชิก | ใช้ `memberCode` | ใช้ login/support/back-office |

## เวอร์ชันย่อสำหรับการตลาด

- คิดคอมจาก `PV`
- `Direct 2 ชั้น 50% + 50%`
- `Team 2 ขา 30%`
- `Team 3 ขา 50%`
- `Matching 2 ชั้น 5% + 5%`
- `Daily cap 10,000 บาท`
- `ครบรอบ 10,000 บาท`
- `ซื้อซ้ำที่มี PV ภายใน 3 วัน` เพื่อเปิดรอบใหม่
- `Cycle cap`: `< 200 PV = 5,000`, `>= 200 PV = 10,000`
- pool gate แรก: `ซื้อเอง + 3 direct + 3 direct buyers`

## ข้อควรระวังเวลาเอาไปสื่อสาร

- อย่านำ `Cashback`, `Unilevel`, `Matrix` ไปใส่ในแผน active ตอนนี้
- อย่าใช้สูตรเก่า `35 USDT = 10 PV`
- อย่าใช้ `memberCode` เป็นรหัสแนะนำสมัคร
