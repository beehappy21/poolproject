## Purpose

This document defines the new matrix rules that replace all legacy matrix behavior.

From this point forward:

- legacy matrix routing rules are not part of the target design
- legacy matrix reentry assumptions are not part of the target design
- implementation should follow only the rules defined in this document and later approved additions

## Locked Rules

### Matrix Structure

- Matrix width remains `2`
- `B1` depth is `3`
- `B2` depth is `2`
- `B3` depth is `2`

Current slot counts by board:

- `B1 = 14` slots
- `B2 = 6` slots
- `B3 = 6` slots

### Matrix Payout Rates

- `B1`
  - `L1 = 0.15`
  - `L2 = 0.15`
  - `L3 = 0.15`
- `B2`
  - `L1 = 0.10`
  - `L2 = 0.10`
- `B3`
  - `L1 = 0.20`
  - `L2 = 0.20`

## Board 1 Open Rule

- A member can open `B1` only after reaching personal volume of `500 PV`
- The open threshold is based on the member's own PV, not team PV
- The first active board is `B1 Round 1`

## Placement Rule

- Matrix placement must follow the real sponsor line as the primary rule
- A point created by a member must stay under that member's sponsor ancestry
- Example:
  - if `A` sponsors `1`
  - and `1` sponsors `1A`
  - then `1A` must be placed inside the matrix line of `1`
- The system must not move that point to an unrelated branch just to fill another open slot

Working meaning for runtime:

- placement is ancestry-bound
- the sponsor tree is the first constraint
- matrix payout may count upward through the ancestor line for the allowed board depth

## Board 1 Payout Scope

- `B1` has `3` payout levels
- The same placement tree is used to calculate payout within those `3` levels
- A point can contribute commission upward for at most `3` ancestor levels inside `B1`

## Board 1 Auto Reorder Holdback

- `B1` commission is withheld at `50%`
- The holdback continues until accumulated withheld amount reaches `750 THB`
- After the accumulated withheld amount reaches `750 THB`, the system stops withholding additional `B1` commission for that reorder target

Display expectation:

- the withheld amount should be shown as `auto reorder`

## Board 1 Completion and Auto Reorder

- `B1` is considered full when all `14` slots are filled
- When `B1` becomes full, the system must check the accumulated `auto reorder` holdback for that member
- If the accumulated holdback has reached `750 THB`, the system must:
  - create a real `Firm` purchase of `500 PV` for that same member account
  - open the next `B1` round as `B1 R+`
- If `B1` becomes full but accumulated holdback is still below `750 THB`, the system must not open the next `B1` round yet
- `B1 R+` means the next sequential round of `B1`
  - example: after `B1 Round 1`, open `B1 Round 2`

Order recording expectation:

- the auto reorder must be recorded as a real product purchase
- the purchase belongs to the `Firm` product category
- the exact product set or SKU bundle will be defined later

## Board 2 Open Rule

- `B2` opens automatically when that member's own `B1` becomes full
- when `B2` opens, it starts with `500 PV` available for board calculation
- `B2` payout can count upward for `2` levels only

## Board 2 Placement Source

- `B2` does not use an unrelated queue
- `B2` uses points generated by members who complete `B1`
- those points must still stay inside the real sponsor line only
- when multiple eligible points are waiting for `B2`, placement priority is based on completion order
  - the member who completes `B1` earlier gets placed into `B2` earlier

## Board 2 Carry-Forward Rule Across B1 Rounds

- if `B1` next round becomes full before the previous `B2` round is full, the system must continue filling the earlier `B2` round first
- this means `B2 Round 1` must be filled to completion before later `B2` points are considered complete for that earlier round
- when `B1 Round 2` becomes full, the system must also open `B2 Round 2`

## Dual Placement Rule For Board 2

When `B1 Round 2` becomes full while `B2 Round 1` is still not full:

- the qualifying point from `B1 Round 2` must continue forward into `B2 Round 1` to help fill the older pending board
- the same qualifying point must also be placed under `B2 Round 2`
- both placements must still stay under the sponsor line within the `B2` structure

Working interpretation:

- one `B1` completion event may create two `B2` placement effects during overlap periods
- one effect is backfill for the older unfinished `B2` round
- one effect is normal placement into the newly opened `B2` round
- this dual placement rule applies only when the earlier `B2` round is still unfinished

## Board 2 Display Rule

- the matrix view should show the point in both `B2` rounds during overlap when dual placement applies
- this is a display and tracking requirement, not proof of double payout by itself
- the member should be able to see:
  - the point continuing inside the earlier unfinished `B2` round
  - the same point appearing under the newer `B2` round

## Board 2 Payout Counting Rule

- when the same source point appears in two `B2` rounds because of overlap, payout counting must treat that source point as `count 1 time`
- the system must not pay duplicated `B2` commission just because the point is displayed in two places
- the duplicated visual placement exists so the board progression can be shown correctly while payout remains single-count

Working payout interpretation from the current design sketch:

- the lower placement pays the first eligible `B2` uplines in that board path
- the upper overlapping display can still show board progression state
- total payable effect from that single source event must still be capped at one counted `B2` event

## Operational Intent

The intended runtime flow for `B1` is:

1. Member reaches personal `500 PV`
2. System opens `B1 Round 1`
3. Downline placements in `B1` must stay aligned to the real sponsor line
4. `B1` payout is calculated upward for up to `3` levels
5. `50%` of `B1` commission is withheld into `auto reorder`
6. Once withheld total reaches `750 THB`, no more `B1` holdback is taken for that reorder target
7. When `B1` becomes full at `14` filled slots, and holdback has already reached `750 THB`, the system places a real `Firm 500 PV` order and opens the next `B1` round
8. If `B1` becomes full before holdback reaches `750 THB`, the next `B1` round stays closed

The intended runtime flow for `B2` is:

1. Member completes that member's own `B1`
2. System opens `B2 Round 1` automatically
3. The new `B2` board starts with `500 PV` for its calculation base
4. `B2` points come from members who complete `B1`
5. `B2` placement order follows `B1` completion order, but still only inside the sponsor line
6. `B2` pays upward for at most `2` levels
7. If a later `B1` round completes before the earlier `B2` round is full, the system continues backfilling the earlier `B2` round first
8. When that later `B1` round completes, the system also opens the next `B2` round
9. During this overlap, the same qualifying point may be recorded in both the older unfinished `B2` round and the newer `B2` round

## Decisions Still Open

The following rules are not yet locked in this document:

- when and how `B3` opens
- whether `B3` also has rounds
- whether `B2` or `B3` use auto reorder
- whether the missing `B1` holdback can be paid by wallet, cash, or must wait only for later matrix commission
- whether the real `Firm 500 PV` auto reorder should use a dedicated order flag for reporting
- whether `750 THB` holdback and `500 PV` reorder are intentionally different business amounts that should be configured separately
- whether holdback is calculated from gross `B1` commission before other deductions
- whether holdback applies only to `B1` or also to later boards
- whether sponsor-line placement can ever spill when the direct branch is already full
- whether the `500 PV` granted at `B2` open is a virtual matrix calculation amount or must come from a separate business posting
- which exact ancestor path receives the single counted `B2` payout when the same source point is displayed in two `B2` rounds

For current implementation, any item above that has already been resolved in the main commission plan must follow the rule in [commission_main_plan_spec.md](/Users/macbook/poolproject/docs/technical-design/commission_main_plan_spec.md).

## Implementation Guidance

When implementation starts, the code should be rewritten to follow these principles:

- use the sponsor line as the primary placement constraint
- avoid reusing legacy queue, claim, descendant-order, or spill assumptions unless explicitly reapproved
- treat `auto reorder` as explicit persisted business state
- treat `B1` round opening as a consequence of successful reorder completion, not legacy CW reentry
